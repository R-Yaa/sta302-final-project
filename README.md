# sta302-final-project
# STA302 Final Project — Airbnb Boston Price Analysis

**Research Question:** To what extent do property type, capacity, and listing
characteristics explain variation in the log-transformed nightly price of
Airbnb listings in Boston?

## Quick Start

1. Clone the repo
2. Place `listings.csv` in `data/raw/` (get it from the OneDrive link in the
   Quercus submission comment from the proposal)
3. Open `sta302-final-project.Rproj` in RStudio
4. Run this one command in the console:

```r
source("scripts/00_run_all.R")
```

This installs all packages, runs the full pipeline, and saves all figures
and tables to `output/`.

## Final Model

After running the full pipeline the final model is:

```
log(price) ~ bedrooms + bathrooms + guests_included +
             property_type + security_deposit
```

Fitted on **n = 3,400** observations (3,561 after cleaning, 161 influential
observations removed using Cook's D > 4/n threshold).

**host_is_superhost** was dropped — both AIC and BIC stepwise agreed, and
the coefficient was not significant (p = 0.548).

---

## Key Results

| Predictor | % Change in Price | p-value |
|---|---|---|
| +1 bedroom | +30.6% | < 0.001 |
| +1 bathroom | +17.2% | < 0.001 |
| +1 guest included | +8.9% | < 0.001 |
| House vs Apartment | −41.0% | < 0.001 |
| Condominium vs Apartment | +0.6% | 0.872 (n.s.) |
| Other vs Apartment | −11.2% | 0.011 |
| +$1 security deposit | +0.03% | < 0.001 |

**Adjusted R² = 0.3013 | AIC = 5103.43 | F = 210.37 (p < 2.2e-16)**

## Known Issues

- `host_is_superhost` is read as logical TRUE/FALSE by `read_csv()`. All
  scripts handle this with `ifelse(host_is_superhost == TRUE, "t", "f")`.
  Do not change this.
- Do not load `MASS` before `dplyr` — it masks `dplyr::select()`. Scripts
  05 and 06 use `car::powerTransform()` instead of `MASS::boxcox()`.
- Always run scripts with working directory set to the project root
  (opening the `.Rproj` file does this automatically).
